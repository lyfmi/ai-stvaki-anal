## Краткое описание продукта

**AI Анализ Ставок** — сигнальный Telegram-бот для беттинг арбитража трафика.

**Основной функционал для пользователя:**
1. Пользователь отправляет **скриншот спортивного события**
2. Бот делает **AI-анализ**
3. Выдаёт **рекомендацию куда ставить**
4. Показывает **прогнозируемую вероятность захода**
5. Даёт **краткое объяснение** почему так нужно поставить ставку

**Монетизация владельца бота:**
- Перед доступом к анализу бот требует **подписку на канал**
- Требует **регистрацию и первый депозит** по партнёрской ссылке владельца
- Таким образом игрок **закрепляется за арбитражником** в партнёрской программе

## Архитектура оригинального решения

### Компоненты системы

| Компонент | Назначение |
|-----------|------------|
| **Бот А** (основной) | Telegram-бот для пользователей, AI-анализ, воронка, админка |
| **Бот Б** (Send-бот) | Отправляет postback-уведомления от партнёрки в канал отчётов |
| **Канал А** | Публичный/приватный канал — подписка пользователей, контент, инструкции |
| **Канал Б** | Приватный канал «Отчёты» — хранение postback-сообщений о регистрациях и депозитах |
| **config.yaml** | Основные настройки бота |
| **lang.yaml** | Тексты интерфейса на всех языках, промокоды |
| **user.db** | База данных пользователей |
| **Web-App** | Мини-приложение внутри Telegram (активируется админом @bivenmo) |
| **start.cmd** | Скрипт запуска на Windows |

### Оригинальный стек (у GrayScheme)

- **ОС:** Windows 10 / Windows Server 2022 (24/7)
- **Запуск:** `start.cmd`
- **Конфиги:** YAML (`config.yaml`, `lang.yaml`)
- **БД:** SQLite (`user.db`)
- **Лицензирование:** активация через @GraySchemebot по ID машины
- **Web-App:** требует активации у админа

> **Для нашего клона:** стек может отличаться (Linux/Docker, Python/Node, PostgreSQL и т.д.), но **логика работы должна быть идентичной**. Активация/лицензирование **не нужны** — пишем для себя.

## Настройка lang.yaml

- Все тексты интерфейса на **всех языках**
- Обязательно указать **свой промокод** в нужных местах для каждого языка
- Поддержка **добавления/удаления языков** (см. видео langsignalbot421.mp4)
- Новые языки: копировать блок русского, перевести, добавить `Name` и `Smiley/Flag`
- В config.yaml: `EnableLanguages` — список кодов языков для отображения в боте
- Для одного языка без выбора: в квадратных скобках указать только один код (например `[kr]`)

## Postback-система (связь с партнёркой)

### Схема работы

1. Создать **Бот Б** в @BotFather — для отправки отчётов
2. Добавить **Бот А** в Каналы А и Б (как админ)
3. Добавить **Бот Б** в Канал Б (как админ)
4. В postback URL подставить:
   - Токен **Бот Б**
   - **chat_id** = ID **Канала Б**

### Шаблоны postback URL

**Регистрация:**
```
https://api.telegram.org/bot{TOKEN_BOT_B}/sendMessage?chat_id={CHANNEL_B_ID}&text={sub1}
```

**Первый депозит:**
```
https://api.telegram.org/bot{TOKEN_BOT_B}/sendMessage?chat_id={CHANNEL_B_ID}&text={sub1}|Firstdep|{amount}
```

> `{sub1}` — Telegram ID пользователя (передаётся партнёркой)  
> `{amount}` — сумма депозита (название переменной может отличаться в разных партнёрках)

### Пример из статьи (демо-токены, НЕ использовать в проде)

- Регистрация: `https://api.telegram.org/bot6758697366:AAHqzTAXphqRg9XlHiFr2hzQ4cH/sendMessage?chat_id=-1002338203752&text={sub1}`
- Депозит: `https://api.telegram.org/bot6758697366:AAHqzTAXphqRg9XlHiFr2hzQ4cH/sendMessage?chat_id=-1002338203752&text={sub1}|Firstdep|{amount}`

### Логика обработки postback ботом

- Бот А **читает сообщения** в Канале Б
- При получении `{telegram_id}` → отмечает пользователя как **зарегистрированного**
- При получении `{telegram_id}|Firstdep|{amount}` → отмечает **первый депозит**
- После депозита → **открывает доступ** к AI-анализу

## Админ-панель (/admin4)

Команда: `/admin4` (доступна только admin ID из config)

**Функции:**
1. **Рассылка** сообщений всем пользователям
2. **Смена реферальной ссылки** (партнёрская ссылка)
3. **Статистика** по боту
4. **Выдать безлимит** — по user ID или пересланному сообщению
5. **Забрать безлимит**

### Обход проверок для админа (тестирование)

1. Взять postback регистрации, заменить `{sub1}` на свой Telegram ID, открыть в браузере
2. Взять postback депозита, указать свой ID и `{amount}` = 777, открыть в браузере
3. Важно: при ручном вызове **без фигурных скобок**, например: `...&text=1993309130`

## AI-анализ ставок (логика из видеообзора)

1. Пользователь делает **скриншот** события в любой БК
2. Отправляет фото в бот
3. AI анализирует скриншот, определяет доступные события/исходы
4. Выдаёт:
   - **Куда ставить** (конкретный исход)
   - **Почему** (краткое объяснение)
   - **Вероятность захода** (обычно коэфф. 1.8–2.2)
5. Анализ занимает несколько секунд

> Из видео: AI не может предсказать результат на 100%, но выдаёт «полезные» рекомендации. Часть ставок будет в плюсе — это мотивирует пользователя продолжать.
